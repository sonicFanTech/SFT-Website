What you can’t do (on purpose)

These are disabled so the demo stays safe.

No real EXE building

The “Build” and “Compile” actions are simulated. The demo won’t run PyInstaller, won’t bundle files, and won’t create installer or uninstaller executables.

No real file access

The demo should not read/write your disk like the real desktop app would. Any “Browse…” buttons are typically mocked for preview purposes.

No permanent projects (by default)

Some demo versions might store settings in your browser. But it’s not meant as your final project editor. Always expect that refreshing could reset things unless stated otherwise in the demo UI.

No 100% perfect accuracy yet

Labels, spacing, and behavior are still evolving. The demo is here to show the direction and the idea. The real app will be more polished and complete.

How the demo works

A quick explanation so people know what’s “real” and what’s “mocked.”

Interactive UI, simulated outcomes

The demo focuses on interaction: menus open, panels change, pages respond, and settings visually update. Some actions (like “Build installer”) show believable results (progress bars, logs, success messages), but they’re not executing real system tasks.

  • Goal: make the demo feel like the real app workflow.
  • Safe: no writing EXEs, no touching your system.
  • Useful: you can see the planned layout + options.

Local-only demo data

If the demo saves anything, it should be local to your browser (like LocalStorage). There is no account system required for the demo.

  • Usually saved: UI preferences (theme, last tab, toggles).
  • Not saved: real files, real projects, sensitive information.
  • Reset: use “Reset Demo Data” or clear site data in your browser.

Tips & shortcuts

Optional, but makes the demo easier to explore.

Recommended way to test

  • Use a desktop/laptop (mouse + keyboard). Mobile may work but can feel cramped.
  • Try resizing the page — most UI panels should adapt.
  • Click around fast — the demo is meant to be spam-click friendly.
  • If something looks stuck, refresh the page or hit the reset button.

FAQ

Quick answers for people who are trying the demo.

Is this the real Roxzned Setup Designer app?

No — this is a web demo made to preview the UI and planned workflow. The real app is a Python desktop tool.

Does the demo build real installers?

Not in the web demo. Any build/compile steps are simulated so the demo stays safe and runs in a browser.

Will my “project” save?

Some settings may save in your browser, but the demo is not meant as your permanent project editor. Always assume it can reset unless the demo specifically says it supports saving/exporting.

What browsers work best?

Chromium-based browsers (Chrome, Edge) usually work best for app-like web UIs. Firefox often works too. If anything looks weird, try another browser or disable strict privacy blockers for the demo site.
