Roxzned Setup Designer ‚Äî Live Web Demo Help

This is a safe, simulated web demo. It shows how the real Roxzned Setup Designer will feel and work, but it does not install anything on your computer and it does not produce a real EXE installer.
1) What the Live Web Demo is
  • A browser-based ‚Äúpreview build‚Äù of the Roxzned Setup Designer interface.
  • Everything is interactive: you can design pages, place UI elements, set colors/images, wire actions, and preview a simulated installer.
  • Saving/loading projects works using your browser (downloads/upload of a JSON file).
  • ‚ÄúGenerated code‚Äù is a starter Python stub (PySide6) meant to demonstrate what export will look like.
2) Quick start
  1. Create or select a page from the page list.
  2. Add elements by dragging them from the Elements panel onto the canvas (or right‚Äëclick the canvas and choose Add).
  3. Select an element to edit its properties (text, size, colors, etc.).
  4. Optionally wire Actions for Buttons/CheckBoxes to make a working installer flow.
  5. Press F5 or open Tools ‚Üí Run Generated Installer‚Ä¶ to preview the simulated installer.
  6. Use File ‚Üí Save Project to download your project JSON.
3) What you CAN do in the demo (fully working)
  • Page Designer
    • Create pages and switch between them.
    • Set Background Color per page.
    • Set Background Image per page (image picker; stored as a browser URL / data URL depending on selection).
  • Elements / Widgets
    • Add elements by dragging from the Elements list or using right‚Äëclick menu.
    • Select elements, move them (drag), resize them (handles), and nudge with arrow keys.
    • Edit common properties: text, size, position, foreground/background colors, font size, bold, etc.
    • Supported widget types in this demo:
      • Label
      • Button
      • CheckBox
      • EditBox (text input)
      • Image
  • Objects / Element List
    • See a list of all elements on the current page.
    • Click an item to select that element on the canvas.
    • Delete/duplicate/reorder via right‚Äëclick or hotkeys.
  • Right‚Äëclick menus
    • Right‚Äëclick the canvas to add new elements at the cursor location.
    • Right‚Äëclick a widget to duplicate, delete, or change layer order (front/back).
  • Undo / Redo
    • Ctrl+Z Undo
    • Ctrl+Y Redo
  • Saving / Loading projects
    • Save Project downloads a JSON file representing the whole project.
    • Open Project loads a JSON file back into the demo.
    • The demo also keeps a browser auto-save snapshot (so refreshing usually won‚Äôt lose your work).
  • Generate Installer Code
    • The code panel outputs Python that embeds your project data as JSON and loads it safely.
    • No broken JSON booleans (false/true/null) ‚Äî it uses json.loads correctly.
  • Simulated Installer Preview
    • Press F5 or use Tools ‚Üí Run Generated Installer‚Ä¶ to open the installer preview.
    • The preview renders your current page layout and widgets.
    • It updates live while you edit in the designer.
    • Buttons/CheckBoxes are interactive and actions work (details below).
4) Actions (how Buttons/CheckBoxes work)

In the real Roxzned Setup Designer, you‚Äôll be able to connect controls to behaviors. The demo includes the same idea: select a Button or CheckBox and configure an Action in the properties panel.

Button Actions (supported in the demo)
  • Back a page ‚Äî moves to the previous page in the installer preview.
  • Next page ‚Äî moves to the next page in the installer preview.
  • Close installer ‚Äî closes the installer preview window.
  • Open URL ‚Äî opens a URL in a new tab (enter the URL in the Action settings).
  • Start install ‚Äî starts a simulated install (progress + log) and/or jumps to the page with role installing if your project has it.
CheckBox Actions (supported in the demo)
  • Enable/Disable targets ‚Äî pick one or more target Buttons/CheckBoxes. When checked, targets become enabled; when unchecked, targets become disabled.
Tip: This is the easiest way to show off a ‚Äúreal‚Äù flow: build a Welcome page ‚Üí License page (checkbox enables Next button) ‚Üí Installing page ‚Üí Finish page.
5) Background Color & Background Image behavior
  • Background Color applies to the page canvas in the designer and the installer preview.
  • Background Image visually sits on top of the color (so if an image is set, it can ‚Äúhide‚Äù the color underneath).
  • If you don‚Äôt see your background color, clear the background image for that page.
6) What you CAN‚ÄôT do in the demo (limitations)
  • No real EXE output ‚Äî the demo cannot compile to an installer executable.
  • No real install/uninstall ‚Äî the ‚Äúinstall‚Äù step is simulated (progress/log only).
  • No real file copying ‚Äî it does not copy files to Program Files/AppData, create real shortcuts, or touch the registry.
  • No full feature set yet ‚Äî advanced widgets, advanced conditions, scripting, localization, etc. are not implemented here.
  • Browser sandbox ‚Äî file access is limited to what your browser allows (you can pick files, but the demo can‚Äôt freely read/write your disk).
The goal is to show the UI/UX and workflow ‚Äî not to replace the real desktop app.
7) Keyboard shortcuts
  • Ctrl+Z Undo
  • Ctrl+Y Redo
  • Delete Delete selected widget
  • Arrow Keys Nudge selected widget
  • Shift + Arrow Keys Nudge faster (if enabled in your build)
  • F5 Run Generated Installer Preview
  • Esc Close open menus/modals
8) Tips, best practices, and ‚Äúshowcase‚Äù ideas
  • Make navigation buttons yourself and assign actions. This matches how the real program will work.
  • Use a License checkbox to enable a ‚ÄúNext‚Äù button (shows off checkbox actions).
  • Add a ‚ÄúVisit Website‚Äù button with Open URL action (shows off external action).
  • Create an Installing page (role = installing) and wire ‚ÄúInstall‚Äù to Start install action.
9) Privacy / safety
  • The demo runs locally in your browser.
  • It does not upload your files anywhere.
  • Project saving is a normal download of a JSON file.
